# 바둑 AI 추천 프로그램 최종 결과물

이 폴더에는 학습된 바둑 AI 모델을 다시 학습하지 않고 바로 실행하기 위한 파일들이 들어 있습니다.

## 포함 파일

```text
go_colab_run_saved_app.ipynb
go_project_colab.zip
trained_go_model.zip
```

각 파일의 역할은 다음과 같습니다.

| 파일 | 설명 |
|---|---|
| `go_colab_run_saved_app.ipynb` | Google Colab에서 실행하는 노트북입니다. 저장된 모델을 불러오고 웹앱 서버를 실행합니다. |
| `go_project_colab.zip` | 웹앱, 모델 서버 코드, 학습 코드가 들어 있는 프로젝트 파일입니다. |
| `trained_go_model.zip` | 이미 학습된 Residual CNN 모델 파일입니다. 다시 학습하지 않고 바로 사용할 수 있습니다. |

## 실행 방법

### 1. Colab 열기

Google Colab에서 `go_colab_run_saved_app.ipynb` 파일을 업로드하여 엽니다.

### 2. 두 zip 파일 업로드

노트북의 첫 번째 코드 셀을 실행하면 파일 업로드 창이 뜹니다.

이때 아래 두 파일을 업로드합니다.

```text
go_project_colab.zip
trained_go_model.zip
```

### 3. 셀을 위에서부터 실행

노트북의 셀을 위에서부터 순서대로 실행합니다.

노트북은 다음 작업을 자동으로 수행합니다.

1. 프로젝트 파일 압축 해제
2. 학습된 모델 압축 해제
3. 모델 파일 존재 여부 확인
4. Flask 서버 실행
5. Colab에서 접속할 수 있는 웹앱 URL 출력

### 4. 출력된 URL 열기

마지막 셀을 실행하면 다음과 비슷한 URL이 출력됩니다.

```text
https://8000-...colab.dev
```

이 링크를 클릭하면 바둑판 웹앱이 열립니다.

## 프로그램 기능

웹앱에서 사용할 수 있는 기능은 다음과 같습니다.

- `Start Game` 버튼으로 게임 시작
- 흑과 백이 번갈아 착수
- 현재 차례에서 AI가 추천하는 Top-5 착수 위치 표시
- 추천 수를 클릭하면 해당 위치에 바로 착수
- 바둑판 위에 마우스를 올리면 흐릿한 돌로 착수 위치 미리보기
- 착수 기록 표시

## 모델 설명

사용한 모델은 19x19 바둑판 상태를 입력받아 다음 착수 위치를 예측하는 Residual CNN Policy Network입니다.

입력은 다음 4개 채널로 구성됩니다.

1. 현재 차례 플레이어의 돌 위치
2. 상대 돌 위치
3. 빈 교차점 위치
4. 현재 차례 정보

출력은 19x19 바둑판의 361개 위치에 대한 확률 분포입니다.

## 주의 사항

이 노트북은 모델을 다시 학습하지 않습니다.

따라서 실행 시간이 짧고, 다음 두 파일만 있으면 다시 사용할 수 있습니다.

```text
go_project_colab.zip
trained_go_model.zip
```

만약 웹앱 링크에서 `HTTP ERROR 500`이 뜨면 서버가 아직 완전히 실행되지 않았거나 중간에 종료된 것입니다. 이 경우 노트북의 서버 실행 셀을 다시 실행한 뒤 마지막 URL 출력 셀을 다시 실행하면 됩니다.

## 프로젝트 요약

이 프로젝트는 SGF 바둑 기보 데이터를 이용해 현재 바둑판 상태에서 다음 착수 위치를 예측하는 딥러닝 모델을 만드는 것을 목표로 합니다.

처음에는 단순 CNN 모델을 사용했고, 이후 Residual CNN과 바둑판 대칭 augmentation을 적용하여 성능을 비교했습니다. 최종 웹앱은 학습된 Residual CNN 모델을 사용하여 현재 판에서 추천 착수 Top-5를 보여줍니다.
